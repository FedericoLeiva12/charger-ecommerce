    Para comenzar este proceso tenemos que explicar que hace cada entidad y a quien requiere:

    Usuario

    El usuario es una persona que interactua con el e-commerce. Por ahora hay tres jerarquias de usuarios utilizando el mismo modelo[admnis,usuarios e invitados] esta jerarquia responde a la entidad Roles

    InfoUsuario

    El usuario cuenta con una serie datos adicionales que son proporcionados una vez este se registra. Mientras no se registre el usuario no se almacenaran datos personales de ningún tipo.

    Roles

    Corresponde a la unica propiedad de la entidad usuario, en un principio el usuario es seteado como invitado. Luego se define si este sera admin o cliente registrado, debiendo así proporcionar sus datos.

    Ordenes

    Las ordenes son todas las intenciones de compra que un usuario tiene, se almacenan salvo que el usuario la elimine dentro de el layout de carrito por lo que es necesario generar un CRUD de este elemento.
    Un usuario registrado podrá acceder a todas sus ordenes pasadas completadas. y al igual que el invitado podrá ver su orden pendiente. No pueden haber mas de una orden como pendiente por usuario.

    Carrito

    Su funcion es obvia y analoga a la de un carrito en la vida real, si los carritos tuvieran los datos de una compra perfectamente accesibles al comprador.

    Productos

    Los productos se definen por su id y nombre, la id lo enlaza con otras tablas mientras el nombre es el codigo con el cual el usuario accede a ellos. Otra forma en que se acceden a ellos es a traves de sus categorias.

    Descripcion

    Un producto tiene una descripcion, esto es un texto que quien genera la entidad del catalogo le asigna para que el comprador pueda familiarizarse con el producto.

    Categorias

    La categoria ofrece una breve descripcion de las caracteristicas del producto, a veces la caracteristica puede ser autodescriptiva como por ejemplo si dijieramos que zappatilla es una categoria o una mejor practica seria decir que categoria tipo de prenda se describe con zapatilla para una zapatilla. Los productos pueden tener muchas categorías y una categoria puede ser parte de muchos productos, es por esto que para definir categoria lo hacemos en prural y con el uso de una tabla intermedia

    Reviews

    Una review es un puntaje y un comentario que un comprador emite sobre un producto que ha comprado de esta forma alerta a otros compradores de su nivel de satisfaccion con el producto. El producto asociado a sus reviews obtiene todos los comentarios y un promedio del puntaje que recibe
